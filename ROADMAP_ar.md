# 🗺️ خارطة الطريق الكاملة — مشروع كشف اعتلال الشبكية السكري (RetinaScan AI)

مشروع متكامل بمستوى عالمي: من البيانات → النموذج → التفسير (Grad-CAM) → واجهة ويب و API → النشر → المسابقات.

---

## 🎯 الهدف النهائي

نظام ذكاء اصطناعي يصنّف صور قاع العين (fundus) إلى 5 درجات لاعتلال الشبكية السكري (0 إلى 4)، **ويشرح قراره** عبر خرائط حرارية (Grad-CAM) تبيّن وين بصّ النموذج. هذا الشيء (الـ explainability) هو اللي بيخلّي المشروع "عالمي" ويلفت أنظار المُوظِّفين ولجان المسابقات، لأن أغلب المشاريع الطلابية بتوقف عند الدقة فقط.

---

## 💻 وين تشتغل؟ (مهم جداً)

ما تحتاج تشتري GPU. هاي الخيارات مرتبة بالأفضلية:

**1) Kaggle Notebooks — الأفضل لهذا المشروع ✅**
- GPU مجاني (T4 ×2 أو P100) حتى ~30 ساعة بالأسبوع.
- الأهم: **بيانات APTOS موجودة جاهزة على Kaggle** — ما تحتاج تنزّل ولا ترفع شي. تضيفها للنوتبوك بضغطة.
- المسابقات نفسها على Kaggle، فبتتعلم البيئة اللي رح تتنافس فيها.
- ابدأ من ملف `notebooks/01_kaggle_quickstart.md` الموجود بالمشروع.

**2) Google Colab — خيار ممتاز كمان**
- GPU مجاني (T4)، وColab Pro رخيص لو بدك جلسات أطول.
- بتحتاج ترفع البيانات على Google Drive أو تستخدم Kaggle API لتنزيلها.

**3) جهازك المحلي** — فقط لو عندك كرت NVIDIA بذاكرة 8GB+. غير هيك خليك على Kaggle.

> **توصيتي:** درّب على **Kaggle**، وارفع الكود على **GitHub**، واعرض الـ demo عبر **Streamlit** (مجاني).

## 📅 الخطة الزمنية (6 أسابيع — قابلة للتسريع)

### الأسبوع 1 — الأساس والفهم
- اقرأ عن اعتلال الشبكية السكري والدرجات الخمسة (بتكفي ساعة قراءة).
- أنشئ حساب Kaggle، وادخل على مسابقة *APTOS 2019 Blindness Detection*.
- شغّل ملف `01_kaggle_quickstart.md` وخلّي pipeline يدور لآخره ولو بدقة منخفضة (المهم كل شي يشتغل end-to-end).
- **مخرج الأسبوع:** نوتبوك يدرّب نموذج بسيط وينتج توقّع.

### الأسبوع 2 — البيانات والمعالجة
- استكشاف البيانات (EDA): توزيع الكلاسات، عيّنات من كل درجة.
- طبّق معالجة Ben Graham (موجودة في `src/preprocess.py`) وقارن الصور قبل/بعد.
- اشتغل على عدم توازن الكلاسات (No-DR مهيمن) — الكود فيه class weights جاهزة.
- **مخرج:** صور EDA + توضيح أثر المعالجة.

### الأسبوع 3 — النموذج الأساسي (Baseline)
- درّب EfficientNet-B4 بدقة 384px.
- راقب المقياس الرسمي **QWK (Quadratic Weighted Kappa)** مش الدقة بس.
- احفظ أفضل checkpoint، اطبع confusion matrix و classification report (`src/evaluate.py`).
- **هدف واقعي:** QWK ≈ 0.88–0.90.

### الأسبوع 4 — التفسير (الميزة المميزة)
- شغّل Grad-CAM (`src/gradcam.py`) على عيّنات من كل درجة.
- تأكد إن النموذج فعلاً بيركّز على الآفات (نزيف، إفرازات) مش على حواف الصورة.
- جهّز صور heatmap حلوة للـ README والبوست.
- **مخرج:** مجموعة صور Grad-CAM مقنعة.

### الأسبوع 5 — رفع المستوى (للمسابقات)
- **Ensemble:** درّب نموذج ثاني (ConvNeXt) واجمع نتائج softmax للموديلين.
- **TTA (Test-Time Augmentation):** متوسّط التوقّعات على قلب/تدوير الصورة.
- **5-Fold CV:** قسّم البيانات 5 طيّات وامتوسّط — بيرفع الثبات.
- (اختياري قوي) ضمّ بيانات EyePACS 2015 (~88 ألف صورة) للتدريب.
- **هدف:** QWK ≈ 0.91–0.92+.

### الأسبوع 6 — النشر والتوثيق
- شغّل Streamlit (`app/app.py`) وانشره على Streamlit Community Cloud (مجاني).
- جهّز REST API بـ FastAPI (`app/api.py`) — يبيّن إنك تعرف الـ production.
- أكمل الـ README بالنتائج والصور.
- انشر بوست LinkedIn (جاهز في `docs/linkedin_post.md`).
- **مخرج:** رابط demo شغّال + repo مرتب + بوست.
